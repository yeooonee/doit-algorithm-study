# 셸 정렬
- 멀리 있는 항목끼리 비교하다, 점점 가까운 값끼리 비교하는 정렬
- 작은 값이 뒤에 있으면 한칸씩 옮기기 때문에 느림. 때문에 처음부터 큰 보폭으로 비교해서 멀리 있는 값도 한번에 옮김

# 퀵 정렬
- 기준값(피벗)을 잡고, 작으면 왼쪽 크면 오른쪽으로 나누기.
- 나뉜 그룹에서 다시 정렬 반복 (재귀)
- 시간 복잡도는 (n log n), 최악의 경우 O(n제곱)

# 병합 정렬
- 배열을 쪼갠다음, 합칠 때 순서대로 끼워 맞추는 것
- __가장 작은 값까지 쪼개서 재귀적으로 병합 정렬?__

# 힙 정렬
- `heapq` 라이브러리 사용
- 힙의 특성을 이용하여 정렬
- 힙 : 부모의 값이 자식의 값보다 항상 크다.or 작다.
- 최댓값을 찾아서 맨 뒤로 보냄.

# 도수 정렬
- `collections.Counter` 로 해결 가능
- 숫자별로 개수 세고, 개수만큼 출력
