# 재귀 (Recursion)

- 재귀 : 어떠한 이벤트에서 자기 자신을 포함하고, 다시 자기 자신을 사용하여 정의되는 것
- 재귀 호출 : 자신과 똑같은 함수를 호출하는 것
<br/>

## 재귀 예시

- **팩토리얼** : `5! = 5*4*3*2*1`, 표준 라이브러리 `math` 모듈의 `factorial()` 함수 제공
- **유클리드 호제법** : 두 정숫값의 최대 공약수를 재귀적으로 구함

<br/>

## 직접 재귀 vs 간접 재귀

- **직접 재귀** : 자신과 똑같은 함수를 호출하는 방식 
- **간접 재귀** : `a()` 함수가 `b()` 함수를 호출하고, 다시 `b()` 함수가 `a()` 함수를 호출하는 방식 

<br/>

## 순수한 재귀 함수

재귀 호출을 **여러 번** 실행하는 함수를 **순수한 재귀**라고 함

```python
def recur(n: int) -> int:
    if n > 0:
        recur(n - 1)
        print(n)
        recur(n - 2)

x = int(input('정수값을 입력하세요.: '))
recur(x)
```

<br/>

#### recur(4) 호출 흐름

```
recur(4)

│   ├── recur(2)
│   │   ├── recur(1)
│   │   │   ├── recur(0)  → 종료
│   │   │   ├── print(1)
│   │   │   └── recur(-1) → 종료
│   │   ├── print(2)
│   │   └── recur(0)      → 종료
│   ├── print(3)
│   └── recur(1)
│       ├── recur(0)      → 종료
│       ├── print(1)
│       └── recur(-1)     → 종료
├── print(4)
└── recur(2)
    ├── recur(1)
    │   ├── recur(0)      → 종료
    │   ├── print(1)
    │   └── recur(-1)     → 종료
    ├── print(2)
    └── recur(0)          → 종료
```

## 재귀 알고리즘 분석 방법

### 1) 하향식 분석 (Top-down)

- 가장 위쪽 함수 호출부터 시작하여 **계단식**으로 자세히 조사해 나가는 방법

### 2) 상향식 분석 (Bottom-up)

- 아래쪽부터 **쌓아 올리며** 분석하는 방법

<br/><br/>

## 재귀 알고리즘의 비재귀적 표현

### 1) 꼬리 재귀 제거

함수 맨 끝의 재귀 호출(꼬리 재귀)은 `while` 루프로 간단히 제거할 수 있다.

```python
def recur(n: int) -> int:
    while n > 0:
        recur(n - 1)
        print(n)
        n = n - 2
```

> **한계** : `recur(n-1)` 은 `n` 출력 **전에** 실행되어야 하므로, 단순 제거가 어렵다.  
> 예를 들어 `n = 4` 일 때, `recur(3)` 처리가 완료될 때까지 `4` 를 어딘가에 저장해야 하는 문제 발생. <br/>
> → **스택(Stack)** 으로 해결 가능!

<br/>

### 2) 스택으로 재귀 제거

```python
from stack import Stack  

def recur(n: int) -> int:
    s = Stack(n)

    while True:
        if n > 0:
            s.push(n)        
            n = n - 1
            continue
        if not s.is_empty():
            n = s.pop()
            print(n)
            n = n - 2
            continue
        break

x = int(input('정수값을 입력하세요.: '))
recur(x)
```

> 스택을 활용해 `recur(n-1)` 실행 전 현재 `n` 값을 저장하고,  
> 나중에 꺼내 출력함으로써 재귀 없이 동일한 동작을 구현한다.
