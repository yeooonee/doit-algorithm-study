# 해시법
- 데이터를 저장할 위치 = 인덱스 를 간단한 연산으로 구하는 것
- 해시값 : 해시 함수의 출력값 (넓은 의미), **배열 인덱스로 쓰는 최종 숫자**. 몇 번 칸에 저장할 지 알려줌 (좁은 의미)
- 해시 테이블 : 해시값을 인덱스로 하여 원소를 새로 저장한 배열 
- 해시 함수 : 키를 해시값으로 변환하는 과정
- 버킷 : 해시 테이블에서 만들어진 원소 

<br/>

- 충돌 미발생 시 시간 복잡도 O(n) 
- 해시 테이블이 너무 크면 메모리 낭비됨. 
- 충돌 피하려면 : 
    1. 해시 함수 <= 해시 테이블 
    2. 소수 선호 


# 해시 충돌
- 저장할 버킷이 중복되는 현상 

## 체인법
- 해시값이 같은 원소를 연결 리스트로 관리 (오픈 해시법)
- 각 버킷에 저장하는 것은 인덱스를 해시값으로 하는 연결 리스트의 앞쪽 노트를 참조 

### 연결 리스트
- 노드 : 데이터 + 포인터
- 데이터 : 실제로 저장하고 싶은 값
- 다음 노드의 주소(포인터) : 다음 노드의 주소 
- python(list, deque), Java(LinkedList)

## 오픈 주소법
- 충돌이 발생했을 때 재해시 하여 빈 버킷을 찾는 방법
- 선형 탐사법
- 버킷에 속성 부여
