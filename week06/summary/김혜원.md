# 큐(Queue)

스택과 같이 데이터를 임시 저장하는 자료구조  
가장 먼저 넣은 데이터를 가장 먼저 꺼내는 **선입선출(FIFO, First In First Out)** 구조

- **인큐(enqueue)** : 큐에 데이터를 추가하는 작업
- **디큐(dequeue)** : 큐에서 데이터를 꺼내는 작업
- **프런트(front)** : 데이터를 꺼내는 쪽
- **리어(rear)** : 데이터를 넣는 쪽

<br/>

## 우선순위 큐(Priority Queue)

인큐할 때 데이터에 우선순위를 부여하여 추가하고,  
디큐할 때 우선순위가 가장 높은 데이터를 먼저 꺼내는 방식이다.

- 파이썬에서는 `heapq` 모듈을 사용하여 구현 가능

<br/>

## 링 버퍼로 큐 구현

### 주요 변수

- `que` : 큐의 배열, 데이터를 저장하는 `list`형 배열
- `capacity` : 큐의 최대 크기
- `front`, `rear` : 맨 앞 원소와 맨 끝 원소를 나타내는 인덱스
- `no`
  - 큐에 쌓여 있는 데이터 개수
  - 비어 있으면 `0`
  - 가득 차 있으면 `capacity`

<br/>

## 주요 메서드

- `__len__()` : 큐에 들어 있는 데이터 개수를 반환
- `is_empty()` : 큐가 비어 있는지 확인 (비어 있으면 `True`, 아니면 `False`)
- `is_full()` : 큐가 가득 차 있는지 확인 (가득 차 있으면 `True` 아니면 `False`)
- `enqueue()` : 큐에 데이터를 인큐 (큐가 가득 찬 경우 `FixedQueue.Full` 예외 발생)
- `dequeue()` : 큐의 맨 앞 데이터를 디큐하여 반환 (큐가 비어 있으면 `FixedQueue.Empty` 예외 발생)
- `peek()` : 맨 앞 데이터를 확인 (다음 디큐 시 꺼낼 데이터를 들여다봄)
- `find()` : 특정 값(`value`)이 들어 있는 위치를 검색 (`front`부터 선형 검색 수행)
- `count()` : 특정 데이터의 개수를 셈
- `__contains__()` : 큐에 데이터가 포함되어 있는지 확인 (포함되어 있으면 True, 아니면 False)
- `clear()` : 큐의 전체 원소 삭제
- `dump()` : 큐의 전체 데이터를 출력 (맨 앞 → 맨 끝 순서대로 출력)


### 인덱스 계산식
```python
(i + front) % capacity
```

## 덱(Deque)

덱은 맨 앞과 맨 끝 양쪽에서 데이터를 삽입 및 삭제할 수 있는 자료구조
큐와 스택을 합친 형태와 유사함
