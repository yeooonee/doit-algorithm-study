# 문자열 검색

어떤 문자열 안에 특정 문자열이 들어있는지 검사하고, 있다면 위치를 찾는 작업

## 1. 브루트 포스법 (단순법)

선형 검색을 단순 확장한 알고리즘<br/> 
앞에서부터 한 글자씩 비교하다가 틀리면 처음부터 다시 비교한다.

```python
def bf_match(txt: str, pat: str) -> int:
    pt = 0
    pp = 0
    while pt != len(txt) and pp != len(pat):
        if txt[pt] == pat[pp]:
            pt += 1
            pp += 1
        else:
            pt = pt - pp + 1
            pp = 0
    return pt - pp if pp == len(pat) else -1
```

- 시간복잡도: O(n), 단순하지만 빠르게 동작
- 검사했던 결과를 버리고 매번 새로 비교한다는 단점이 있음

### 파이썬 내장 문자열 검색 도구

| 도구 | 설명 |
|---|---|
| `ptn in txt` | txt에 ptn이 포함되어 있는지 (True/False) |
| `ptn not in txt` | 포함되어 있지 않은지 |
| `str.find(sub)` | 가장 작은 인덱스 반환, 없으면 -1 |
| `str.rfind(sub)` | 가장 큰 인덱스 반환, 없으면 -1 |
| `str.index(sub)` | find와 동일하나 없으면 ValueError |
| `str.rindex(sub)` | rfind와 동일하나 없으면 ValueError |
| `str.startswith(prefix)` | prefix로 시작하면 True |
| `str.endswith(suffix)` | suffix로 끝나면 True |

## 2. KMP법

브루트 포스법과 달리 검사했던 결과를 버리지 않고 재활용하는 알고리즘<br/> 
미리 패턴 자체를 분석해 "건너뛰기 표(skip table)"를 만들어두고, 불일치가 발생해도 처음부터 다시 비교하지 않는다.

```python
def kmp_match(txt: str, pat: str) -> int:
    pt = 1
    pp = 0
    skip = [0] * (len(pat) + 1)

    skip[pt] = 0
    while pt != len(pat):
        if pat[pt] == pat[pp]:
            pt += 1
            pp += 1
            skip[pt] = pp
        elif pp == 0:
            pt += 1
            skip[pt] = pp
        else:
            pp = skip[pp]

    pt = pp = 0
    while pt != len(txt) and pp != len(pat):
        if txt[pt] == pat[pp]:
            pt += 1
            pp += 1
        elif pp == 0:
            pt += 1
        else:
            pp = skip[pp]

    return pt - pp if pp == len(pat) else -1
```

- 시간복잡도: 최악의 경우에도 O(n)
- 다만 구현이 복잡하고, 반복되는 패턴이 없는 경우엔 큰 효율 이득이 없음

## 3. 보이어-무어법

패턴을 **뒤(끝)에서부터** 비교하며, 불일치하는 문자가 나오면 미리 만든 건너뛰기 표를 참고해 포인터를 한 번에 크게 이동시키는 알고리즘<br/>
텍스트의 모든 문자를 다 보지 않아도 되기 때문에 가장 빠른 문자열 검색 알고리즘이다.

```python
def bm_match(txt: str, pat: str) -> int:
    skip = [None] * 256

    for pt in range(256):
        skip[pt] = len(pat)
    for pt in range(len(pat)):
        skip[ord(pat[pt])] = len(pat) - pt - 1

    pt = len(pat) - 1
    while pt < len(txt):
        pp = len(pat) - 1
        while txt[pt] == pat[pp]:
            if pp == 0:
                return pt
            pt -= 1
            pp -= 1
        pt += skip[ord(txt[pt])] if skip[ord(txt[pt])] > len(pat) - pp \
            else len(pat) - pp

    return -1
```

- 시간복잡도: 최악 O(nm), 평균 O(n/m) — 패턴 길이(m)가 길수록 평균적으로 더 빠름
- 문자열 길이가 0~255(ASCII 범위)일 때를 가정한 구현

## 알고리즘 비교

| 알고리즘 | 정의                           | 평균/최악 시간복잡도        |
|---|------------------------------|--------------------|
| 브루트 포스법 | 처음부터 한 글자씩 단순 비교             | O(n)               |
| KMP법 | 이전 비교 결과를 재활용해 되돌아가지 않음      | 최악에도 O(n)          |
| 보이어-무어법 | 패턴 끝에서부터 비교, 불일치 시 큰 폭으로 건너뜀 | 평균 O(n/m), 최악 O(n) |
