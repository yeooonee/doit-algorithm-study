# 큐
- 선입선출(FIFO) 방식
- 가장 먼저 넣은 데이터를 가장 먼저 꺼냄. <br/>

| 용어 | 설명 |
|------|------|
| 인큐 | 데이터 추가 |
| 디큐 | 데이터 꺼냄 |
| 프런트 | 데이터 꺼내는 쪽 |
| 리어 | 데이터 넣는 쪽 |

### 우선순위 큐 
- 인큐할 때 데이터에 우선순위 부여하여 추가
- 디큐할 때 우선순위가 가장 높은 데이터를 꺼내는 방식
- heapq 모듈에서 제공. 

## 링 버퍼 
- 디큐할 때 배열 안의 원소를 옮기지 않는 큐 
- 배열 맨 끝의 원소 뒤에 맨 앞의 원소가 연결됨.
- 어떤 원소가 앞(front)이고 뒤(rear)인지 변수가 식별.
- 오래된 데이터를 버리는 용도로 활용할 수 있음. 

### 양방향 대기열 덱
- 맨 앞과 맨 끝 양쪽에서 데이터를 넣고 꺼낼 수 있는 자료구조
- collections.deque
